L'ordre d'éxécution du code par le navigateur

Par défaut, un navigateur va lire et exécuter le code dans l'ordrede son écriture

Lorsque le navigateur arrive à un script, il va stopper le traitement du reste du HTML jusqu'a ce que le code JavaScript soit chargé dans la page et exécuter.

Résumons ce qu'il se passe dans ce cas

  1. Le navigateur commence à analyser (ou à traiter le code HTML
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et télécharger (si il est en externe) puis exécuter. Durant tout ce temp l'analyseur du navigateur bloque l'affichage du HTML ce qui peut ralentir, si le script est long, l'affichage de la page ;
  4. Dès que le JS a été exécuter, le contenu HTML est affiché

Ce souci d'attente est résolu grâce au téléchargement asynchrome des données avec l'attribut async ou defer dans le casou l'on fait appel à des fichiers externes

Le point virgule

Le point virugule ; est utiliser pour signifier la fin d'une instruction (une variable, une condition, une boucle etc.).
